# Music Player (C++ Terminal Music Player)

یک پخش‌کننده‌ی موسیقی متنی (TUI) که با C++17 و کتابخانه‌ی تک‌فایلی
`miniaudio` نوشته شده است. کد کاملاً شیءگرا (OOP) است و هر کلاس در فایل‌های
`.h`/`.cpp` جداگانه قرار دارد.

```
Music_Player/src/miniaudio.h
```

ماکروی `MINIAUDIO_IMPLEMENTATION` فقط و فقط در `src/Player.cpp` تعریف شده است،
پس نیازی به هیچ کار اضافه‌ای ندارید.

---

## ساخت و اجرا

از ریشه‌ی پروژه (پوشه‌ای که `Data/` و `Makefile` در آن است):

```bash
make          # کامپایل
./music_player # اجرا  (یا: make run)
make clean    # پاک‌سازی فایل‌های ساخته‌شده
```

کامپایلر با `-std=c++17 -Wall -Wextra` و بدون هیچ هشداری بیلد می‌شود.
برنامه مسیرها را به‌صورت ثابت و نسبت به ریشه‌ی پروژه می‌خواند، پس حتماً از
ریشه‌ی پروژه اجرا کنید (نه از داخل `src/`).

## ساختار پوشه‌ها

```
Music_Player/
├── Makefile
├── README.md
├── src/                # همه‌ی کلاس‌ها (به‌علاوه‌ی miniaudio.h که باید اضافه کنید)
└── Data/
    ├── library.csv     # کاتالوگ آهنگ‌ها
    ├── settings.cfg    # وضعیت ذخیره‌شده
    ├── Playlists/      # فایل‌های .m3u
    └── Musics/         # فایل‌های صوتی نمونه (.wav)
```

> داده‌های داخل `Data/` فقط نمونه هستند تا برنامه بلافاصله قابل اجرا باشد؛
> می‌توانید آن‌ها را با داده‌های خودتان جایگزین کنید. فرمت CSV:
> `title,artist,album,genre,year,duration_sec,file_path`

## معماری و کلاس‌ها

| لایه | کلاس‌ها |
|------|---------|
| مدل داده | `Song`, `Playlist`, `MusicLibrary` |
| بارگذاری/ذخیره | `CsvLoader`, `M3uLoader`, `ConfigManager` |
| موتور پخش | `Player` (ماشین حالت + miniaudio) |
| رابط کاربری | `UIRenderer`, `InputHandler`, `Screen` (پایه‌ی انتزاعی) و ۶ صفحه |
| هماهنگ‌کننده | `Application`, `AppContext`, `main` |

### پوشش اصول شیءگرایی
- **کپسوله‌سازی:** همه‌ی فیلدها `private` و دسترسی فقط از طریق getter/setter.
- **وراثت:** `Screen` (انتزاعی) → ۶ صفحه‌ی مشتق.
- **چندریختی:** فراخوانی `render()`/`handleInput()` از طریق اشاره‌گر `Screen*`
  و **مخرب مجازی**.
- **انتزاع:** `render()` به‌صورت `pure virtual`.
- **مدیریت حافظه‌ی پویا:** `new`/`delete`؛ مالکیت آهنگ‌ها فقط با `MusicLibrary`
  است (تنها جایی که `Song` آزاد می‌شود). `Playlist` آهنگ‌ها را حذف نمی‌کند.
- **File I/O:** `ifstream`/`ofstream` برای CSV، M3U و settings.
- **STL:** `vector` و `map`.
- **مدیریت خطا:** `try/catch` روی I/O، اعتبارسنجی ورودی و ادامه بدون کرش.

## امکانات

- **صفحه‌ها:** Main Menu، Now Playing، Playlists، Browse (مرتب‌سازی/جستجو/فیلتر)،
  Filter، Settings.
- **حالت‌های پخش:** `NO_REPEAT`, `REPEAT_ONE`, `REPEAT_ALL`, `SHUFFLE`.
- **مرتب‌سازی:** بر اساس عنوان/خواننده/آلبوم/سال/مدت (با `+` برای نزولی، مثل `4+`).
- **جستجو:** بدون حساسیت به بزرگی/کوچکی حروف روی عنوان، خواننده و آلبوم.
- **فیلتر:** بر اساس خواننده یا آلبوم، همراه با شمارش هر مقدار.
- **ماندگاری وضعیت:** پلی‌لیست فعال، حالت پخش و آخرین آهنگ در `settings.cfg`.
- **پایداری:** فایل صوتی گم‌شده/CSV خراب باعث کرش نمی‌شود؛ خطا گزارش و رد می‌شود.

### امکانات امتیازی پیاده‌سازی‌شده
- **Seek نسبی** (`seekBy`) برای جابه‌جایی ±۱۰ ثانیه (آماده برای اتصال به کلیدهای جهت).
- **افزودن آهنگ به پلی‌لیست** (`Playlist::addSong`).

## نکته درباره‌ی صدا

پخش صدا از طریق `miniaudio` انجام می‌شود. اگر برنامه روی محیطی بدون کارت/دستگاه
صوتی اجرا شود، موتور صدا به‌صورت بی‌صدا کار می‌کند ولی همه‌ی منطق برنامه و ماشین
حالت بدون کرش ادامه می‌یابد (degradation تدریجی).

---

## عیب‌یابی صدا (برنامه اجرا می‌شود ولی صدا ندارد)

اگر برنامه بدون کرش اجرا می‌شود ولی صدایی پخش نمی‌شود، به ترتیب این‌ها را بررسی کن:

1. **پیام هنگام اجرا را ببین:** اگر پیام
   `‎[audio] could not initialise the audio engine` را دیدی، یعنی سیستم‌عامل هیچ
   دستگاه صوتی در دسترس برنامه نگذاشته و حالت «بی‌صدا» فعال شده است. در صفحه‌ی
   Now Playing هم خط `! Audio device unavailable - silent mode` نشان داده می‌شود.

2. **محیط اجرا (شایع‌ترین علت):** در WSL، اتصال SSH، کانتینر (Docker)، یا ماشین
   مجازی بدون کارت صدا، هیچ backend صوتی وجود ندارد. روی دسکتاپ واقعی
   (ویندوز/مک/لینوکس با محیط گرافیکی) صدا باید پخش شود.
   - **WSL2:** پخش صدا نیاز به WSLg (ویندوز ۱۱) دارد؛ روی ترمینال ویندوز تست کن.

3. **سرویس صدای لینوکس:** مطمئن شو PipeWire/PulseAudio/ALSA فعال است
   (`pactl info` یا `aplay -l`). پخش مستقیم یک فایل برای تست:
   `aplay Data/Musics/midnight_city.wav`.

4. **فرمت/مسیر فایل:** miniaudio فرمت‌های wav/mp3/flac را پشتیبانی می‌کند. اگر
   پیام `Could not decode audio file` دیدی، مسیر داخل `library.csv` یا `.m3u`
   اشتباه است؛ مسیرها نسبت به ریشه‌ی پروژه حساب می‌شوند، پس حتماً با
   `./music_player` از ریشه اجرا کن (نه از داخل `src/`).

5. **لینک روی لینوکس:** Makefile به صورت خودکار روی لینوکس کتابخانه‌های
   `-lpthread -lm -ldl` را لینک می‌کند (miniaudio برای ALSA به این‌ها نیاز دارد).

> در هر حالت، نبود صدا باعث توقف یا کرش برنامه نمی‌شود؛ این یک انتخاب طراحی است
> (degradation تدریجی) تا برنامه روی هر محیطی قابل اجرا باشد.
